# 币安Gate仓位同步项目 - 深度记忆文档

## 项目概述
实现币安交易所和Gate.io交易所的BTC合约仓位同步，检测异常自动修复。

## 创建时间: 2026-05-07
## 最后更新: 2026-05-08

---

## 一、任务清单

### 1. 同步任务 (binance_gate_sync.py)
- **功能**: 币安→Gate仓位同步
- **轮询间隔**: 2秒
- **同步模式**: 净方向（多空相抵后同步）
- **杠杆**: 20x 逐仓
- **无止盈止损**
- **自动修复**: 检测到宕机自动重启

### 2. 健康检查任务 (health_check.py)
- **功能**: 检查同步任务运行状态
- **检查间隔**: 5分钟
- **检查项目**:
  - 币安API状态
  - Gate API状态
  - 持仓获取
  - 轮询执行
  - 同步一致性
- **自动修复**: 检测到任务宕机自动重启

## 五、同步逻辑

```
币安净持仓 = 多仓BTC - 空仓BTC
Gate目标合约 = 币安净持仓 × 10000

示例:
- 币安多0.001 + 空0.001 = 净0 → Gate跳过
- 币安多0.002 + 空0.001 = 净多0.001 → Gate开多10合约
- 币安多0.001 + 空0.002 = 净空0.001 → Gate开空10合约
```

## 六、自动修复逻辑

健康检查任务每5分钟执行一次，发现问题自动修复：

1. 检测同步任务进程不存在 → 自动重启
2. 检测轮询日志停止更新(>60秒) → 自动重启
3. 检测API异常 → 记录告警

## 九、注意事项

1. Gate逐仓模式不允许同合约同时多空，所以同步净方向
2. IOC订单需要带price参数
3. 币安positionAmt是BTC数量，不是手数
4. 健康检查任务也会被自动修复（如果宕机）
